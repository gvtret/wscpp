# План реализации легковесного WebSocket-стека на C++

## Общие требования
- C++11 или выше
- Система сборки: CMake
- Поддержка семантической версионности
- Полная документация (пользовательская и разработческая)
- Git-репозиторий на GitVerse
- Примеры клиента и сервера
- Сравнительный анализ с существующими библиотеками (C++11-совместимыми)
- Юнит, регрессионные и интеграционные тесты

## Технологический стек
- OpenSSL для TLS/SSL
- standalone ASIO (beast или asio) для сетевых операций
- CMake 3.10+ для сборки

---

## Атомарные этапы реализации

### Этап 1: Инициализация репозитория
**Цель**: Создать Git-репозиторий и настроить базовую структуру

**Действия**:
1. Инициализировать Git-репозиторий в `wscpp`
2. Создать файл `.gitignore` для C++ проектов
3. Создать базовую структуру директорий:
   - `src/` — исходный код библиотеки
   - `include/` — заголовочные файлы
   - `tests/` — тесты
   - `examples/` — примеры клиентов/серверов
   - `docs/` — документация
   - `scripts/` — скрипты сборки и генерации
4. Создать `README.md` с кратким описанием проекта
5. Создать `LICENSE` файл (MIT или Apache-2.0)
6. Создать `VERSION` файл для хранения текущей версии (например, "0.1.0")

**Критерии готовности**:
- Репозиторий инициализирован
- Базовая структура директорий создана
- README и LICENSE существуют
- Git-репозиторий готов к работе

**Критерии готовности**:

- CMake генерирует работоспособный Makefile/VS проект

### Этап 3: Интеграция standalone ASIO
**Цель**: Подключить ASIO как зависимость и создать базовый сетевой интерфейс

**Действия**:
1. Создать `cmake/FindASIO.cmake` или использовать `FetchContent` для подключения ASIO
2. Добавить `asio` и `asio/ssl` в CMake конфигурацию
3. Создать минимальный тест `test_asio.cpp` для проверки работоспособности ASIO
4. Создать базовую обёртку `wscpp::net::asio_socket` для абстракции ASIO сокетов
5. Добавить ASIO в `CMakeLists.txt` как зависимость

**Критерии готовности**:
- ASIO подключен и компилируется
- Базовая обёртка сокета работает
- Тест ASIO проходит успешно

---

### Этап 4: Интеграция OpenSSL
**Цель**: Подключить OpenSSL для поддержки TLS/SSL

**Действия**:
1. Создать `cmake/FindOpenSSL.cmake` или использовать `find_package(OpenSSL)`
2. Добавить OpenSSL в CMake конфигурацию
3. Создать минимальный тест `test_openssl.cpp` для проверки работоспособности OpenSSL
4. Создать обёртку `wscpp::crypto::ssl_context` для управления SSL-контекстами
5. Добавить OpenSSL в `CMakeLists.txt` как зависимость

**Критерии готовности**:
- OpenSSL подключен и компилируется
- Базовая обёртка SSL-контекста работает
- Тест OpenSSL проходит успешно

### Этап 5: Реализация базового WebSocket-фрейма
**Цель**: Реализовать базовую логику разбора и формирования WebSocket-фреймов

**Действия**:
1. Создать `wscpp::frame::parser` для разбора входящих фреймов
2. Создать `wscpp::frame::builder` для формирования исходящих фреймов
3. Реализовать поддержку базовых opcode (CONTINUATION, TEXT, BINARY, CLOSE, PING, PONG)
4. Реализовать маскирование данных (client-to-server)
5. Добавить юнит-тесты для фреймов (`test_frame.cpp`)

**Критерии готовности**:
- Базовый парсер фреймов работает
- Базовый сборщик фреймов работает
- Юнит-тесты проходят

---

### Этап 6: Реализация WebSocket-соединения (базовая)
**Цель**: Создать базовый класс WebSocket-соединения

**Действия**:
1. Создать `wscpp::connection` — базовый класс для WebSocket-соединений
2. Реализовать асинхронные операции read/write через ASIO
3. Реализовать handshake (поддержка как обычного, так и TLS)
4. Добавить обработку CLOSE-фреймов
5. Добавить юнит-тесты для соединения (`test_connection.cpp`)

**Критерии готовности**:
- Базовое соединение работает
- Handshake проходит успешно
- Юнит-тесты проходят
